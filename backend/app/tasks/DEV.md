# Tasks 模块 DEV 文档

## 1. 文档用途说明
本文档用于维护 `app/tasks/` 下可独立运行的脚本任务，明确其与 `services/` 的调用关系，避免在任务脚本中堆积本属于服务层的业务实现。

## 2. 项目模块总览
| 文件 | 主要职责 |
|---|---|
| `app/tasks/run_log_producer.py` | 循环生成模拟日志并通过 Kafka producer 发送 |

## 3. 模块职责边界
- 应该放在这里：CLI/脚本入口、调度循环、进程级参数、组合调用已有 service。
- 不应该放在这里：重复实现 `log_generator` 或 `producer` 内部协议、HTTP 接口、schema 定义。

## 4. 已实现功能清单
- 已实现基于 `settings.log_producer_interval_seconds` 的定时发送循环。
- 已组合 `build_mock_log` 与 `send_log_message`。

## 5. 待开发功能清单（P0-P3）
- P0：增加启动失败时的明确错误输出（Kafka 不可达等）。
- P1：支持命令行参数（发送条数、间隔、topic 覆盖）。
- P2：优雅退出与发送计数指标打印。
- P3：与容器/运维脚本对齐的 health 自检子命令。

## 7. 禁止重复实现清单
| 能力 | 正确位置 | 禁止行为 |
|---|---|---|
| 模拟日志构造 | `app/services/simulation/log_generator.py` | 禁止在 `run_log_producer.py` 内复制粘贴生成逻辑 |
| Kafka 发送 | `app/services/kafka/producer.py` | 禁止在 task 中直接 new 另一套 producer 协议实现 |
